CrystalClear AI Audio Enhancer — Project Working & Flow
===========================================================

1) What this project is
------------------------
A Flutter app that lets the user:
- Select an audio file (Input)
- (Optionally) Record audio using the microphone
- Play/pause/stop the input audio
- Send the input audio to an external “enhancement” API
- Receive the enhanced audio bytes
- Write the enhanced result to a temporary WAV file
- Load & play the enhanced audio
- Save/download the enhanced WAV file

Main entry point: lib/main.dart


2) App startup (main → MaterialApp)
-----------------------------------
- main() calls:
  - runApp(const AudioEnhancerApp())

- AudioEnhancerApp (StatelessWidget) builds:
  - MaterialApp(theme: dark, home: AudioEnhancerScreen())

So the whole UI and logic live inside AudioEnhancerScreen.


3) Screen state (AudioEnhancerScreen)
-------------------------------------
AudioEnhancerScreen is a StatefulWidget that manages:

A) Input file / recording
- _currentFile : PlatformFile? (selected or recorded input)
- _recordingPath : String? (temporary recording path)

B) Playback of input or enhanced audio
- _player : AudioPlayer (from just_audio)
- _isPlaying : bool
- _currentPosition : Duration
- _totalDuration : Duration

C) Enhancement request
- _isEnhancing : bool
- _progress : double (0..1)
- _cancelToken : CancelToken? (for cancelling in-progress request)
- _enhancedFile : PlatformFile? (temporary WAV created from API response)

D) Waveform preview UI
- _waveformData : List<double>
- _isWaveformLoading : bool

E) Recording animation indicator
- _animationController / _animation for “REC” UI


4) initState: stream subscriptions
----------------------------------
During initState(), the app wires audio streams to UI state:
- _player.playerStateStream → updates _isPlaying
- _player.positionStream → updates _currentPosition
- _player.durationStream → updates _totalDuration
- _player.processingStateStream:
  - if ProcessingState.completed → reset playhead and seek to 0

It also starts a repeating animation controller for recording indicator.


5) Input audio workflow
-------------------------

5.1 Selecting an audio file: _pickFile()
- Calls FilePicker.platform.pickFiles(type: FileType.audio)
- On selection:
  - Stops any current playback: _player.stop()
  - Resets enhancement-related state:
    - _enhancedFile = null
    - waveform + duration reset
  - Loads input audio into just_audio:
    - await _player.setFilePath(_currentFile!.path!)
  - Generates waveform preview:
    - await _generateWaveformData(_currentFile!.path!)


5.2 Recording audio: _toggleRecord() → _startRecording() / _stopRecording()
- _toggleRecord():
  - if already recording → _stopRecording()
  - else → _startRecording()

- _startRecording():
  - requests microphone permission (permission_handler)
  - uses record package:
    - getTemporaryDirectory()
    - builds a path recording_<timestamp>.m4a
    - _recorder.start(encoder: aacLc, bitRate, sampleRate, path)
  - sets _isRecording = true

- _stopRecording():
  - finalizes recording with _recorder.stop()
  - if a valid file exists:
    - stops playback
    - creates a PlatformFile representing the recorded file (m4a)
    - clears previous enhancement result
    - loads it into the player:
      - await _player.setFilePath(path)
    - generates waveform preview


5.3 Playing input audio: _togglePlay() / _stopPlayback()
- _togglePlay():
  - if _isPlaying → _player.pause()
  - else:
    - if neither _currentFile nor _enhancedFile is set → show “No audio loaded”
    - else → _player.play()

- _stopPlayback():
  - _player.stop()
  - seek to zero
  - reset _isPlaying and _currentPosition


6) Waveform preview workflow
------------------------------
Waveform generation is handled by _generateWaveformData(filePath).

Important note:
- It currently does a simple/simulated waveform generation.
- It reads file bytes only to access context (and then uses a fixed-seed Random(42)).
- It generates 100 random doubles as placeholder waveform bars.

The waveform painter (WaveformPainter) then:
- Draws bars across the width
- Colors “played portion” bars green (based on currentPosition/totalDuration)
- Draws a white playhead line while playing


7) Enhancement workflow (the core feature)
-------------------------------------------
Triggered by the UI button: _enhance()

7.1 Preconditions
- If _currentFile == null:
  - shows SnackBar: “Please select an audio file first”
  - returns

7.2 UI state before sending
- sets:
  - _isEnhancing = true
  - _progress = 0

7.3 Network call setup
- _cancelToken = CancelToken()
- dio = Dio()

7.4 Build multipart/form-data request
- formData = FormData.fromMap({
    'file': MultipartFile.fromFile(_currentFile!.path!, filename: _currentFile!.name)
  })

7.5 POST to enhancement API
- endpoint constant:
  - _apiUrl = 'https://abid1012-audio-enhancement.hf.space/direct-enhance'

- dio.post<List<int>>( _apiUrl, ... ) with:
  - responseType: ResponseType.bytes
  - cancelToken: _cancelToken!
  - onSendProgress and onReceiveProgress:
    - update _progress = sent/total or received/total

7.6 Handle response
- If response.data != null:
  - bytes = response.data!
  - writes to temporary WAV:
    - tempDir = await getTemporaryDirectory()
    - enhancedPath = tempDir.path + '/enhanced_<timestamp>.wav'
    - File(enhancedPath).writeAsBytes(bytes)
  - sets _enhancedFile with:
    - path, size, name
  - shows SnackBar: “Enhancement complete!”

7.7 Error handling and cleanup
- catch(e): show SnackBar “Enhance failed: …”
- finally():
  - _isEnhancing = false

7.8 Cancelling an enhancement: _cancelEnhance()
- calls _cancelToken?.cancel()
- sets _isEnhancing = false
- shows SnackBar: “Enhancement cancelled”


8) Post-enhancement workflow
-------------------------------

8.1 Loading enhanced audio: _loadEnhancedAudio()
- if _enhancedFile == null → return
- stops playback
- loads enhanced file path into player:
  - await _player.setFilePath(_enhancedFile!.path!)
- regenerates waveform preview for enhanced file
- shows SnackBar: “Enhanced audio loaded”

The UI play/pause controls are the same buttons as input playback.


8.2 Saving/download enhanced output: _saveToDownloads()
- if _enhancedFile == null → return

Android:
- requests storage permission (permission_handler)
- downloadsDir logic:
  - tries Directory('/storage/emulated/0/Download')
  - fallback to getExternalStorageDirectory()

iOS/others:
- uses getApplicationDocumentsDirectory()

Then:
- fileName = enhanced_audio_<timestamp>.wav
- copies the temp file to downloadsDir path
- shows SnackBar success/failure


9) UI layout summary (what the user sees)
-------------------------------------------
The build() method shows two cards:
- Input Audio card (left on wide screens / stacked on narrow)
  - Select File button
  - Record/Stop button
  - Play/Pause and Stop playback buttons
  - Waveform preview

- AI Enhancement card
  - Enhance Audio button
  - Cancel button while enhancing
  - Progress bar + percentage while enhancing
  - When _enhancedFile is ready:
    - “Load & Play” button
    - “Save” button


10) Where to change behavior
-----------------------------
- Enhancement endpoint URL:
  - lib/main.dart: AudioEnhancerScreen._apiUrl

- Playback:
  - uses just_audio AudioPlayer

- Recording:
  - uses record AudioRecorder

- Waveform preview:
  - _generateWaveformData() and WaveformPainter
